# Учет остатков материалов

Приложение для учета ликвидных остатков ЛДСП, МДФ, столешниц и других материалов на мебельном производстве.

## Возможности

- **Сканирование QR-кодов** — USB/Bluetooth сканером или камерой телефона
- **Защита от дублей** — каждый QR-код имеет уникальный ID
- **База остатков** — поиск, фильтрация, группировка
- **Управление статусами** — В наличии, Зарезервирован, Использован, Списан
- **История действий** — все изменения записываются
- **Экспорт данных** — Excel и CSV
- **Резервное копирование** — сохранение базы данных
- **Пользователи** — роли: Администратор, Оператор, Мастер

## Установка

### 1. Установка Node.js

Требуется Node.js версии 18 или выше. Скачайте с [nodejs.org](https://nodejs.org/).

### 2. Установка зависимостей

Откройте командную строку в папке проекта и выполните:

```bash
npm install
```

### 3. Создание базы данных

```bash
npm run db:push
npm run db:generate
npm run db:seed
```

### 4. Запуск приложения

```bash
npm run dev
```

Приложение будет доступно по адресу: **http://localhost:3000**

## Использование

### Сканирование на компьютере

1. Откройте страницу **Сканирование** (/scan)
2. Отсканируйте QR-код USB-сканером
3. Данные автоматически попадут в поле ввода
4. Нажмите Enter или кнопку "Добавить остаток"

### Мобильное сканирование

1. Откройте приложение на телефоне в той же сети
2. Перейдите на страницу **Мобильное** (/mobile-scan)
3. Нажмите "Начать сканирование"
4. Наведите камеру на QR-код

### Ручное добавление

1. Перейдите на страницу **Добавить** (/add)
2. Заполните форму
3. Нажмите "Добавить остаток"

### Поиск и фильтрация

1. Откройте **База** (/leftovers)
2. Используйте фильтры для поиска
3. Кликните на остаток для просмотра карточки

### Изменение статуса

1. Откройте карточку остатка
2. Нажмите "Изменить статус"
3. Выберите новый статус
4. Для "Зарезервирован" и "Использован" укажите номер заказа

### Экспорт данных

1. Откройте **Отчеты** (/reports)
2. Нажмите "Экспорт в Excel" или "Экспорт в CSV"
3. Файл скачается автоматически

### Резервное копирование

1. Откройте **Настройки** (/settings)
2. Нажмите "Создать резервную копию"
3. Файл базы данных скачается автоматически

## Формат QR-кода

Рекомендуемый формат — JSON:

```json
{
  "id": "OST-2026-000001",
  "orderNumber": "2458",
  "materialType": "ЛДСП",
  "materialName": "Egger U708",
  "color": "Светло-серый",
  "thickness": 16,
  "length": 1200,
  "width": 450,
  "quantity": 1,
  "createdAt": "2026-05-13",
  "comment": "Остаток после распила кухни"
}
```

Альтернативный формат — строка с разделителями:

```
OST-2026-000001|2458|ЛДСП|Egger U708|Светло-серый|16|1200|450|1|2026-05-13|Комментарий
```

## Создание тестового QR-кода

### Вариант 1: Онлайн-генератор

1. Откройте [qr-code-generator.com](https://www.qr-code-generator.com/)
2. Вставьте JSON-строку
3. Скачайте QR-код
4. Распечатайте и отсканируйте

### Вариант 2: Генерация в приложении

1. Откройте **Настройки**
2. В разделе "Генератор QR" введите данные
3. Скачайте сгенерированный QR-код

## Доступ к приложению в локальной сети

### На компьютере

1. Узнайте IP-адрес компьютера:
   - Windows: `ipconfig` в командной строке
   - Найдите "IPv4 Address" (например, 192.168.1.100)

2. Запустите приложение:
   ```bash
   npm run dev -- -H 0.0.0.0
   ```

3. Откройте на телефоне: `http://192.168.1.100:3000`

### На телефоне

1. Подключитесь к той же Wi-Fi сети
2. Откройте браузер
3. Введите адрес компьютера с портом 3000

## Проверка защиты от дублей

1. Отсканируйте QR-код с существующим ID
2. Приложение покажет сообщение "Этот QR-код уже есть в базе"
3. Откроется карточка существующего остатка
4. Повторное добавление будет заблокировано

## Техническая поддержка

При возникновении проблем:

1. Проверьте, что Node.js установлен
2. Убедитесь, что зависимости установлены: `npm install`
3. Пересоздайте базу данных: `npm run db:push`
4. Проверьте логи в консоли
